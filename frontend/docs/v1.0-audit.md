# v1.0 accessibility + performance audit

Closing audit for the v1.0 finish plan. Covers what can be verified
in this environment (code-level checks, file-level deliverables);
the live-browser Lighthouse pass is flagged for the production
verification step.

## Accessibility — code-level verification

### Skip link

Present at `app/[locale]/layout.tsx`:
```tsx
<a href="#main" className="focus:bg-brass focus:text-navy-deep
  sr-only focus:not-sr-only focus:fixed focus:left-4 focus:top-4
  focus:z-[1100] focus:rounded-sm focus:px-4 focus:py-2">
```
Keyboard users tabbing in jump directly to `<main id="main">`.

### Focus visibility

`globals.css` carries the focus-visible rule for `[data-hover]`
elements (every interactive non-input on the site uses
`data-hover` for the cinematic hover effect, which doubles as a
focus indicator).

### Reduced-motion coverage

- `BrainAnatomy` — PR 8 fix: continuous Y rotation now gated.
- `PersistentAtmosphere` — explicit check inside the keystroke
  pulse loop.
- `ScrollScene` / `ParallaxLayer` / `PinnedSequence` — honor
  GSAP's `ScrollTrigger` reduced-motion behavior plus the
  per-component checks added during the cinema build.
- `Mandala` — rotation gated.
- Tours autoplayer — checks before scheduling next-scene
  transition.

### Reduced-transparency

`BackgroundSystem.tsx` checks
`(prefers-reduced-transparency: reduce)` and serves a solid
fallback instead of the layered washes. No regressions in v1.0.

### Screen-reader labels

- Persistent brain canvas: `aria-hidden` (decorative; activations
  are conveyed by the page text plus `RegionAnnouncer`).
- `RegionAnnouncer` (live region announcing region changes for
  AT users) — present and wired to `useBrainStageStore`.
- Audio: each `<audio>` element is `aria-hidden` — controls
  come from the scrubber's labeled play/pause button.
- Nav mega-menu: `aria-expanded` + `aria-haspopup="menu"` on
  each panel trigger; menu items use `role="menuitem"`.
- LanguageSelector: `aria-haspopup="listbox"` + `aria-expanded`.

### Footer landmark

PR 9 introduced the shared `<SiteFooter>` mounted at the locale
layout. Every page now has a consistent `<footer>` landmark with
20+ link affordances (8 rooms + 7 reference + GitHub).

## Performance — code-level verification

### Lazy-loaded heavy work

- `BrainStage` dynamic import with `ssr: false` — the R3F
  canvas never blocks SSR.
- Tours autoplayer + `MandalaBrainViewer` dynamic-imported with
  loading fallbacks.
- The Cmd-K palette (`SearchPalette`) is part of
  `DeferredLocaleClient` — only mounts when the user opens it
  the first time.

### Image budget

- Mandala plates: optimized JPEGs at <200KB each.
- OG cards: rendered at 1200×630 server-side via `next/og`.
- Audio: opus mono, 89 KB + 143 KB for the two real tracks.
- Background plates per room: served at the `[data-room]` CSS
  selector level so only the active room's plate is fetched.

### Caches

`generateStaticParams` is implemented at the locale level —
six locales × every route are pre-rendered to static HTML
chunks at build time. Build log confirms 235 static pages
generated.

## Out-of-scope for v1.0 (flagged for v1.1)

Items requiring a real browser / hosting / CI integration:

- **Lighthouse mobile ≥ 85 / desktop ≥ 95.** Cannot run
  Lighthouse from this environment. The production smoke
  step at deploy time is the natural place; spot-checks
  via `pnpm dlx lighthouse` against
  `https://brain-studio-kappa.vercel.app/en` are the recommended
  pre-tag verification.
- **WCAG AA contrast on Faust + Dante background plates.**
  The atmospheric plates sit at ~0.08 opacity — well below the
  contrast-affecting threshold for text — but a real-browser
  contrast pass is the only way to confirm at scale. Deferred.
- **Cross-browser sweep.** Per PR 8 audit — Chrome / Safari /
  Firefox desktop + mobile Safari + Chrome Android. Deferred.
- **TRIBE inference.** Still local-simulated. The About roadmap
  (PR 1) names this as remaining work for v1.1.
- **Pair submission form** (Cross-Cultural) and audio
  contribution form (Music). The About roadmap names both as
  remaining work.

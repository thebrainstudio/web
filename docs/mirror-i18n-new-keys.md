## Translation register notes

The site's existing prose is in a **formal-literary register** —
restrained, hedged, no exclamations. New strings should match:

- **th** — formal Thai (ระดับเป็นทางการ). Anatomy keeps Latin script;
  function phrases translate. See `messages/th.json` for the existing
  voice in `essays`, `regions`, `fieldNotes`.
- **es / ca** — formal Castilian Spanish / Catalan. The home page uses
  the polite formal (no `tú`). Anatomy keeps clinical Latin where the
  Spanish-speaking neurology literature does (e.g. *giro frontal
  inferior*).
- **ja** — Japanese with brain-anatomy terms typically kept in
  English/Latin OR transliterated as ヘシュル回 etc. Match the
  existing convention in `messages/ja.json` regions block.
- **zh-CN** — formal Mandarin. Anatomy uses standard medical terms
  (布罗卡区, 海马, etc.).

## How to apply

1. Translator opens `messages/<locale>.json` for each non-EN locale.
2. Adds the new keys under the `mirror.caption.*`, `mirror.inspector.*`,
   `mirror.pinnedCard.*`, `mirror.brainViews.*` namespaces.
3. The caption generator + components need a small refactor to read
   from i18n instead of hardcoding EN — that's a follow-up patch on
   the `mirror-improvements` branch (or a new branch named
   `mirror-i18n`).

## Validation

Run `pnpm run build` from `frontend/`. next-intl will warn if any
locale's JSON shape doesn't match `en.json`. Use that as the parity
check before merging the translator's PR.

# Mirror known issues — honest punch list

> The Mirror page after the `mirror-improvements` PR is far richer
> than it was, but it's not polished-polished. This document is the
> honest list of rough edges that any reviewer will find. Written so
> we don't pretend the system is perfect.

## TRIBE inference

### Auditory regions dominate text-only predictions

**Symptom:** the top 4 regions are almost always
`hg_left`, `hg_right`, `pstg_left`, `pstg_right` regardless of input
content. Memory text, math text, fear text — all produce
auditory-heavy top regions.

**Why:** TRIBE v2 was trained trimodal on movie audio transcripts.
With audio + video set to zero, the model defaults to grounding
language in the speech-perception regions (Heschl's gyrus +
posterior STG) because that's where its training signal came from.
The regional differences across inputs are visible *within* the
top-4 ranking shifts, but the absolute regions don't change.

**Mitigation:** a text-only baseline correction would surface
semantic variation. Approach:
1. Run TRIBE on ~20 diverse text prompts at deploy time.
2. Compute the mean prediction → "typical text-only output."
3. For new predictions, subtract this baseline before z-scoring.
4. Surface "deviation from typical text-only" instead of raw output.

This is ~90 min of research-grade work and is not yet in the PR.

### TRIBE off-distribution caveat is surfaced honestly

The attribution chip below the textarea reads
"text-only, average-subject" when TRIBE is online. The disclaimer
line below the caption also says "Predicted activation. Not your
brain." The user is told what they're looking at.

## Caption generator

### Caption body is English-only

The caption that appears below the textarea after a settled prediction
is generated by `lib/mirror/caption-generator.ts` from a curated
20-region `FUNCTION_PHRASES` table — all English strings. The
disclaimer below the caption IS translated across all 6 locales, but
the caption sentences themselves render in English on `/th`, `/es`,
`/ca`, `/ja`, `/zh-CN`.

See [`mirror-i18n-new-keys.md`](./mirror-i18n-new-keys.md) for the
exact key set translators need to fill in.

### Caption can phrase awkwardly for very short inputs

The signal-threshold check hides the caption when there's no real
signal (`top-1 activation < 0.35`), but inputs just above the
threshold sometimes produce captions like *"The model recruited
Heschl's gyrus, the posterior STG, and the angular gyrus. Primary
hearing meeting word recognition meeting abstract reading."* — which
is correct but reads abruptly. A "warming up" sentence would soften
this. Future patch.

### Example comparison rarely fires

The `mirror.savedExamples` cosine-similarity check (≥ 0.92 to
mention an example) almost never fires with text-only TRIBE because
the auditory bias makes all predictions cosine-close to each other.
The comparison sentence works correctly in the embedding baseline
mode but is mostly silent when TRIBE is on.

## Tunnel + backend

### Tunnel URL is ephemeral

Cloudflare quick tunnels (`https://*.trycloudflare.com`) lose their
URL on `cloudflared` restart. Mitigation in this PR: the new
`/api/tunnel` discovery endpoint + `publish_tunnel.py` lets the
backend self-publish the current URL on each fresh tunnel. Frontend
reads at request time.

**Remaining limitation:** the discovery endpoint's in-memory cache
is per-edge-instance and survives only one Vercel cold-start. For
multi-region reliability, swap to `@vercel/kv` (paid feature) or use
a Cloudflare-managed named tunnel + a stable domain. Both are 30-min
follow-ups; neither is in this PR.

### First request after a sleep takes ~30–60 s

The local TRIBE backend pre-loads Llama-3.2-3B + the TRIBE checkpoint
into VRAM at uvicorn startup (Phase C.2 in this PR), so the FIRST
request to a freshly-started backend is fast. But when the user's
laptop sleeps and the backend stays running on a sleep-resumed
process, Llama gets evicted from VRAM and the next request triggers
a re-load. That re-load can take ~30–60 s. Frontend falls back to
the Vercel embedding-baseline during the wake-up window, so the
user sees a less-accurate prediction during that window.

### No tunnel auth — anyone with the URL can hit the predictor

The local backend's rate limiter (30 req/min, 1000/day per IP) is
the only gate. For a portfolio site this is acceptable, but if the
URL is shared in public it could be abused. A shared-secret header
check between the Vercel frontend and the local backend would
tighten this — `Phase C.4` in the punch list, not yet built.

### No graceful CPU fallback

If CUDA is unavailable on the backend host, the TRIBE engine fails
to load and the engine cascade falls all the way through to the
embedding baseline. There's no "CPU TRIBE" path — Llama-3.2-3B on
CPU would take 30+ s per inference, which is unusable. Documented
in `phase-10-recon.md` §10.0.4.

## Mirror UI

### Hover-coupled regions use the impressionistic predictor, not TRIBE

The hover word-to-region coupling shows which regions the
*impressionistic* (lexical) predictor would have pushed toward —
not TRIBE's per-word contributions, which TRIBE doesn't expose
(TRIBE gives one whole-input prediction). Surfaced honestly in the
section disclaimer.

### Region pills use English region names

`MirrorInspector.tsx` uses `regionById[id].displayName` directly. The
displayName values are translated in `messages/<locale>.json` under
`regions.<id>.displayName`, but the inspector doesn't call into
next-intl for these — it just uses the canonical English from
`lib/regions.ts`. Translator follow-up needed.

### Pinned card placement may collide with other affordances

The pinned card lives `fixed bottom-6 left-6` on desktop. On the
Mirror page this works because no other component lives there. If
Move 4 is ever extended to other rooms, placement will need
re-thought.

### Touch / mobile: word hover-via-tap is single-active-at-a-time

Tapping a word swings the brain toward that word's regions; tapping
another word swaps. There's no "clear all" affordance — the user
either taps a different word or scrolls away. Acceptable for the
inspect-a-prediction flow but not a polished interaction.

## PNG export

### Brain in the PNG is schematic, not photorealistic

The fingerprint uses a top-down peanut-ellipse silhouette with 20
colored dots. The dots are placed at `regions[i].position` (the
stylized atlas-nav coordinates), NOT the real anatomical positions
from the fsaverage5 mesh. This is acceptable because:
- The PNG is rendered server-side via Satori, which can't run WebGL.
- The dots' positions are well-curated and read as a brain pattern.
- The composition is meant as a "museum wall card", not a clinical
  scan.

A future patch could pre-render the 4 anatomical views from
`BrainViews.tsx` as static SVGs and embed those instead.

### Font fallback to Georgia

`ImageResponse` doesn't embed Fraunces by default. The PNG uses
Georgia as the serif fallback. Visually close but not identical to
the live page. Embedding Fraunces would require fetching the font
binary at request time — defer to a future patch.

## Site-wide gaps

### Sister rooms don't have the attribution chip

The Mirror page's attribution chip (TRIBE engine state + license
URL) is currently Mirror-only. Music, Cross-Cultural, Threshold,
Archetypes, About, Cellular have no equivalent. They don't run
TRIBE inference yet so the chip would mostly say "baseline only";
the cleaner move is to wait until those rooms have a meaningful
interaction.

### Cellular View is barely sketched

Per the brief, Cellular View should have a 1400 ms atmosphere fade,
60 s neuron rotation, 2 s synapse animation. Current implementation
is a single neuron rotation; nothing else. Not in this PR.

### Search palette is English-only

`Cmd+K` opens the palette with English placeholder text and category
labels. The actual search index is locale-aware but the chrome
isn't. Not in this PR.

### No analytics-driven user testing

Vercel Analytics + Speed Insights are wired (Phase D.5) but no
data-driven changes have been made yet. After a week of real
traffic there will be data on where users drop off in the Mirror
flow; those insights should feed a Move 1.5 / Move 2.5 patch.

## Build hygiene

### Lint baseline has pre-existing errors

`pnpm run lint` reports 53 errors + 15 warnings, all dating to before
this PR (React Compiler advisories on WebGL/animation hotspots; a
few legacy patterns in `app/error.tsx`; unused-vars cleanup). This
PR deliberately did not refactor those.

### Type checking is clean

`tsc --noEmit` runs as part of `pnpm run build` and passes on every
commit on this branch.
